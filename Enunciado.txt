Descripción del ProyectoDiseñar e implementar un Sistema Experto (SE) basado en el conocimiento para la identificación de especies animales, utilizando el lenguaje Python, el motor de reglas Py-Rete y la biblioteca gráfica PyQt. El sistema debe emular el razonamiento deductivo de un experto en biología, permitiendo al usuario ingresar evidencias y obteniendo una clasificación taxonómica precisa.2. Requerimientos Funcionales (RF)RF 01: Base de Conocimientos EstructuradaEl sistema debe contener reglas de producción que abarquen al menos 20-25 especies, incluyendo fauna emblemática de Colombia (Chigüiro, Cóndor, Oso de Anteojos, Jaguar) y especies globales populares (Jirafa, Cebra, Elefante).Las reglas deben estar organizadas jerárquicamente:Nivel 1: Clase (Mamífero, Ave, Reptil, Anfibio, Pez).Nivel 2: Orden/Dieta (Carnívoro, Herbívoro, etc.).Nivel 3: Especie (Nombre común específico).RF 02: Motor de Inferencia (Forward Chaining)Utilizar el algoritmo RETE (vía Py-Rete) para el encadenamiento hacia adelante.El sistema debe procesar "Hechos" (Facts) ingresados por el usuario y disparar las reglas cuyas condiciones se cumplan, generando nuevos hechos derivados hasta alcanzar una conclusión.RF 03: Interfaz Gráfica Reactiva (PyQt)Control de Evidencia: La interfaz debe presentar opciones (Categorías: Morfología, Hábitat, Alimentación).Lógica de Bloqueo: Al seleccionar una característica, la interfaz debe consultar al motor (o a una lógica de exclusión) para deshabilitar (gray out) opciones imposibles.Ejemplo: Si el usuario marca "Tiene Plumas", las opciones "Tiene Escamas" o "Tiene Branquias" deben bloquearse automáticamente.Botón "Identificar": Ejecuta la inferencia final con los hechos recolectados.RF 04: Módulo de Trazabilidad y ExplicaciónTras la identificación, el sistema debe desplegar una ventana o panel de "Log de Razonamiento".Debe listar las reglas que se activaron y por qué.Formato: "Identificado como: Jaguar. Razón: Es Mamífero (tiene pelo) + Es Carnívoro (come carne) + Tiene manchas + Vive en selvas americanas."3. Requerimientos No Funcionales (RNF)RNF 01 - Modularidad: La base de conocimiento (reglas) debe estar separada del código de la interfaz gráfica para facilitar actualizaciones.RNF 02 - Usabilidad: La interfaz en PyQt debe ser intuitiva, utilizando iconos o colores para diferenciar las categorías de los animales.RNF 03 - Robustez: El sistema debe manejar casos de "Inferencia Incompleta". Si los datos no son suficientes para llegar a una especie, debe informar el grupo taxonómico más cercano alcanzado.4. Estructura de Reglas sugerida para LaTeXPara tu informe, puedes definir la lógica de encapsulamiento de esta manera:$$\text{Regla\_Jaguar: } \text{IF } (\text{Clase: Mamífero}) \wedge (\text{Hábitat: Selva}) \wedge (\text{Piel: Manchas Anulares}) \rightarrow \text{ASSERT } (\text{Especie: Jaguar})$$Tips adicionales para tu desarrollo:Encapsulamiento en Py-Rete: Recuerda definir tus Facts heredando de Fact y usar el decorador @Rule para las producciones.Comunicación PyQt <-> Py-Rete: Te sugiero crear una clase "Controladora" que maneje una instancia del KnowledgeEngine. Cada vez que el usuario haga clic en un checkbox, actualizas los hechos en el motor y pides una validación de qué reglas siguen siendo posibles para bloquear el resto en la UI.
